**Security and Access**

Authentication

C-Auth-1.
customerId определяется системой из JWT-токена.

SR-Auth-1.
Система не должна принимать customerId из пользовательского ввода.

Authorization (RBAC)

SR-AuthZ-1.
Роль customer может создавать только один отзыв по каждому продукту.

SR-AuthZ-2.
Роли analyst и pm имеют доступ к чтению и аналитике отзывов.

SR-AuthZ-3.
Роль admin имеет полный доступ.

SR-AuthZ-4.
При нарушении прав доступа система возвращает HTTP `403 Forbidden`.

Privacy

SR-PII-1.
customerId маскируется для ролей customer, analyst, pm.

SR-PII-2.
customerId возвращается в явном виде только роли admin.
